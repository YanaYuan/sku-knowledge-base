SKU 解剖图

China Price List 中的 SKU 遵循三种主要结构模式。下面逐一拆解：

Pattern 1: Azure (hyphen-delimited)

Az VM F/FS Srs - F2/F2s - 10 Hrs - US SC
Platform = Azure Service = VM Series = F/FS Size = F2/F2s Qty = 10 Unit = Hours Region = US South Central

Pattern 2: Volume Licensing On-Prem (space-delimited)

SQL Server Enterprise Core ALng LSA 4L
Product = SQL Server Edition = Enterprise Component = Core LangCode = All Languages LicType = License + SA QtyPack = 4-License

Pattern 3: Modern SaaS (space-delimited)

M365 E5 IP & Governance Sub Per User
Suite = M365 Tier = E5 Feature = IP & Governance License = Subscription Unit = Per User

General Patterns

Azure: [Az] [Service] [Series] Srs [Win]-[Size]-[Qty]/[Unit]-[Modifier]-[Region]
On-Prem: [Product] [Edition] [Component] [LangCode] [LicType] [QtyPack] [Qualifier]
SaaS: [Suite] [Tier] [Feature] Sub Per [Unit] [Qualifier]

Modifiers (optional): DvTst = Dev/Test | Win = Windows License | AHB = Azure Hybrid Benefit | EEA = European Economic Area
